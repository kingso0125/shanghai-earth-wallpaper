# iPhone 自动换壁纸

云端发布后，固定地址为：

- Lock：`https://<GitHub用户名>.github.io/<仓库名>/lock.jpg`
- Home：`https://<GitHub用户名>.github.io/<仓库名>/home.jpg`
- 状态：`https://<GitHub用户名>.github.io/<仓库名>/manifest.json`

## 创建快捷指令“更新实时地球”

1. 添加“当前日期”，再添加“格式化日期”，自定义格式为 `yyyyMMddHH`，作为本小时缓存参数。
2. 添加“文本”，内容为 Lock 地址并追加 `?v=` 与上一步的格式化日期。
3. 添加“获取 URL 内容”，URL 使用上一步文本。
4. 添加“设定墙纸照片”，选择锁定屏幕墙纸，将输入设为上一步图片，关闭“运行时显示”。
5. 再添加“文本”，内容为 Home 地址并追加同一个 `?v=` 缓存参数，然后“获取 URL 内容”。
6. 再添加“设定墙纸照片”，选择主屏幕墙纸，关闭模糊与“运行时显示”。
7. 保存为“更新实时地球”，首次运行时允许联网、照片与墙纸权限。

查询参数不会改变云端文件，只用于绕过 GitHub Pages/CDN 与 iOS 的旧图片缓存。

## 每小时自动运行

iOS 的个人自动化没有一个“每小时”循环开关。稳定做法是在“快捷指令 → 自动化”中建立 24 个“特定时间”自动化，从 00:25 到 23:25，每个调用同一个“更新实时地球”快捷指令，并关闭“运行前询问”和“运行时通知”。只需建立一次，图片地址之后保持不变。

云端任务安排在每小时第 17 分钟生成。如果手机自动化使用 `:25`，通常能取得本小时的新图片；若卫星源暂时中断，服务器仍生成壁纸，但 `manifest.json` 会标记为 `cached`，不会伪装为新观测。
